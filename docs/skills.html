Skills & Docs

How skills work, how to author them, and the full catalog.

How skills work

A skill is a folder under skills/ containing at minimum a SKILL.md. Agents use progressive disclosure:

  1. Metadata (name + description) — always visible to the agent (~100 words)
  2. SKILL.md body — loaded when the skill triggers (<500 lines ideal)
  3. Bundled resources — loaded on demand (scripts, references, assets)

This keeps agent context lean. Only the relevant skill content enters the conversation. 

skills/my-skill/
├── SKILL.md          # required — agent instructions
├── scripts/          # optional — runnable helpers
├── references/       # optional — docs loaded on demand
└── assets/           # optional — templates, icons, etc.

SKILL.md format

Every SKILL.md starts with YAML frontmatter: 

---
name: my-skill
description: >
  What this skill does and when to use it.
  Include trigger phrases users would actually type.
  Lean "pushy" — err toward triggering rather than missing.
---

# My Skill

Operational instructions in Markdown.
Keep under 500 lines. Use references/ for overflow.

Adding a skill

Use the scaffold script or copy manually:

# Automated (recommended)
$ uv run scripts/new-skill.py my-skill

# Manual
$ cp -r templates/skill-template skills/my-skill

After creating your skill, add an entry to the skill index below.

Authoring guidelines

  • Description — say what the skill does AND when to trigger it. Be specific. Include realistic trigger phrases. No vague marketing.
  • Instructions — imperative voice, concise. Explain why behind constraints instead of heavy-handed MUSTs.
  • Examples — prefer over explanation when shorter.
  • Scripts — prefer over long natural-language procedures. Python with uv run, TypeScript with bun run.
  • References — use for large docs that don't need to be in context by default. Include a table of contents for files >300 lines.
  • No filler — every sentence must change behavior or understanding.

Conventions

  • Slug format: lowercase, hyphens, no underscores (frontend-design)
  • One folder per skill, always
  • SKILL.md is the only required file
  • Colocate everything inside the skill folder—no cross-references between skills
  • Python scripts: use uv run with inline dependency metadata
  • TypeScript scripts: use bun run
  • No Docker, no heavy build systems
